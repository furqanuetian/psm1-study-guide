# Scrum Glossary — PSM I

## Core Terms

| Term | Definition |
|------|------------|
| **Scrum** | A lightweight framework for solving complex adaptive problems. |
| **Sprint** | A time-boxed period (≤ 1 month) where a usable Increment is created. |
| **Product Owner** | Accountable for maximizing the value of the product. |
| **Scrum Master** | Accountable for establishing Scrum and helping everyone understand it. |
| **Developers** | Professionals who create the Increment each Sprint. |

## The 3 Pillars of Empiricism

| Pillar | Definition |
|--------|------------|
| **Transparency** | Significant aspects of the process must be visible to those responsible. |
| **Inspection** | Scrum artifacts and progress must be inspected frequently. |
| **Adaptation** | If any aspect deviates, adjustments must be made ASAP. |

## The 5 Scrum Values

| Value | Meaning |
|-------|---------|
| **Commitment** | People personally commit to achieving the Sprint Goal. |
| **Courage** | Scrum Team members have courage to do the right thing. |
| **Focus** | Everyone focuses on the Sprint Goal and Product Goal. |
| **Openness** | The Scrum Team and stakeholders agree to be open about work. |
| **Respect** | Team members respect each other's skills and independence. |

## Artifacts & Commitments

| Artifact | Commitment |
|----------|------------|
| Product Backlog | Product Goal |
| Sprint Backlog | Sprint Goal |
| Increment | Definition of Done |

## Events

| Event | Purpose |
|-------|---------|
| **Sprint** | The container for all other events. |
| **Sprint Planning** | Plan the work for the Sprint. |
| **Daily Scrum** | 15-minute inspection for Developers. |
| **Sprint Review** | Inspect the Increment and adapt the Product Backlog. |
| **Sprint Retrospective** | Plan improvements for the next Sprint. |
