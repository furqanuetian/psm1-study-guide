## Key Lessons Learned

| Concept | Correct Understanding |
|---------|----------------------|
| **Daily Scrum attendance** | Only **Developers** are required to attend |
| **Scrum Master's role at Daily Scrum** | Teaches Developers to keep it within 15 mins — does **not** facilitate |
| **Progress management** | **Developers** manage their own progress |
| **Scope negotiation** | PO + Developers can renegotiate scope — Sprint Goal is fixed |
| **Sprint length** | Considers risk, sync, and max timebox — "All of the above" |
| **Self-managing team** | No sub-teams, no handoffs, no stakeholders in Daily Scrum |
| **Business objective** | **Product Owner** owns value and progress communication |
