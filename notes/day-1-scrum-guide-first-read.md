## Key Takeaways

| Topic | My Understanding |
|-------|------------------|
| **Scrum Definition** | A lightweight framework for solving complex problems. Not a methodology — it's a framework. |
| **Accountabilities** | 3 roles: PO (value), SM (process), Developers (work). No "team lead." |
| **Empiricism** | 3 pillars: Transparency, Inspection, Adaptation. Everything builds on these. |
| **Sprint** | Fixed timebox (≤ 1 month). No changes during Sprint. |
| **Events** | Sprint Planning, Daily Scrum, Sprint Review, Sprint Retrospective. |
| **Artifacts** | Product Backlog, Sprint Backlog, Increment. Each has a commitment. |
